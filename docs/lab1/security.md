# Security

계정과 결제 수단을 다루는 만큼 보안 수칙을 철저히 준수해야 합니다. 다음 사항은 Lab 1
온보딩을 수행하는 동안 필수로 고려해야 할 보안 포인트입니다.

## Identity & Access (Entra ID)

- **전용 계정 사용**: 실습 전용 Outlook 계정을 생성해 기존 조직 계정과 분리합니다.
- **다단계 인증(MFA)**: Founders Hub 신청 과정에서 제공되는 이메일 기반 MFA를 반드시 설정하고, 가능하다면 Authenticator 앱으로 확장합니다.
- **로그인 세션 관리**: 프라이빗 창에서 작업하고, 실습 종료 후에는 모든 Microsoft/LinkedIn 세션에서 로그아웃합니다.

## Secrets & Keys management

- **결제 정보 보호**: 본인 확인용 결제 수단을 입력할 때는 공용 PC를 사용하지 말고, 스크린샷을 공유할 경우 카드 번호와 CVC를 가립니다.
- **보안 코드 공유 금지**: MFA 코드, 이메일 인증 코드는 개인이 직접 입력하며, 타인과 공유하지 않습니다.
- **GitHub PAT 관리**: Codespaces 자동화를 위해 토큰을 발급했다면, 최소 권한 원칙을 적용하고 사용 후 즉시 폐기합니다.

## Network & Data protection

- **안전한 네트워크 사용**: 공용 Wi-Fi가 아닌 신뢰할 수 있는 네트워크에서 온보딩을 진행합니다.
- **브라우저 저장 정보 삭제**: 실습 완료 후 브라우저의 임시 파일과 자동완성 데이터를 삭제해 계정 노출을 방지합니다.
- **비정상 활동 모니터링**: Azure 포털의 보안 경고나 로그인 기록에서 의심스러운 활동이 감지되면 즉시 비밀번호를 변경하고 MFA 설정을 재확인합니다.
