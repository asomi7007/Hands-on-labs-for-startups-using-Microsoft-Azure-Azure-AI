# Conversation

Founders Hub 온보딩 과정에서 참가자 질문에 빠르게 대응하기 위해 지원 채널과 대화형
스크립트를 준비해 두면 시행착오를 줄일 수 있습니다. 아래는 실습 중 자주 등장하는
대화 시나리오를 다룰 때 참고할 수 있는 설계 가이드입니다.

## Agent/Chat design

- **단계 기반 답변**: 계정 생성, MFA 설정, 결제 정보 입력 등 각 주요 단계를 기준으로 답변을 나눕니다.
- **상태 확인 질문 우선**: “현재 어느 단계에 계신가요?”, “어떤 화면에서 멈춰 있나요?”와 같이
  사용자의 진행 상황을 먼저 파악할 수 있는 질문을 배치합니다.
- **스크린샷 요구 기준**: 크레딧 배너 미표시, 결제 수단 오류처럼 증거가 필요한 경우에는 필수로
  스크린샷을 요청하고, 민감 정보(카드 번호 등)는 가린 뒤 전달하도록 안내합니다.
- **다국어 고려**: 영문 UI와 국문 UI가 혼재될 수 있으므로, 버튼/메뉴 이름을 양쪽 언어로 병기합니다.

## Prompts & evaluation

- “Azure 크레딧이 보이지 않아요”와 같은 문의에 대해 **확인 절차 체크리스트**(Azure 포털 배너, 비용 관리 화면)를
  바로 제공하도록 준비합니다.
- MFA나 결제 오류는 브라우저 캐시/기기의 영향이 큰 편이므로, **프라이빗 창 재시도**를 기본 대응으로 둡니다.
- 대화 로그를 정리할 때는 `요청 사항 → 제공한 해결책 → 최종 상태` 순으로 기록하여 향후 교육 데이터로 활용합니다.
- 매 실습 세션 후에는 해결된 문의 유형을 분류해 FAQ에 반영하거나, 부족한 스크린샷/설명 자료를 업데이트합니다.

## Tool / RAG flows

- **내부 가이드 검색**: 온보딩 문서(이 저장소의 `docs/lab1` 디렉터리)를 RAG 소스로 연결하여 일관된 답변을 제공합니다.
- **헬프데스크 템플릿**: 결제·계정 오류는 `admin@eldensolution.kr`로 전달할 때 필요한 정보를 자동으로 수집하는 양식을 만듭니다.
- **상태 추적 보드**: 각 참가자의 진행 상태(예: 계정 생성 완료, 크레딧 적용 대기)를 Trello/Planner 등으로 시각화하면
  실시간 지원이 수월합니다.
- **Feedback loop**: 챗봇이나 수동 대응에서 새로 발견한 이슈는 GitHub Discussions나 전용 노션 페이지에 기록해 문서 개선에 반영합니다.
