# Introduction

Microsoft for Startups Founders Hub Lab 1은 새로 만든 Microsoft 계정을 기반으로
LinkedIn, Founders Hub, Azure 구독까지 한 번에 온보딩하여 즉시 사용할 수 있는
$1,000 Azure 크레딧을 확보하는 것이 목표입니다. 이 페이지는 전체 실습이 어디에
집중되어 있는지 한눈에 확인할 수 있도록 개요를 제공합니다.

## Goals

- 개인 및 기존 업무 계정과 충돌하지 않는 **전용 Microsoft/LinkedIn 계정**을 준비합니다.
- Founders Hub `Get started quickly` 흐름을 따라 **$1,000 Azure 크레딧**을 활성화합니다.
- Azure 포털과 비용 관리 메뉴에서 **크레딧 적용 여부를 검증**하는 체크포인트를 숙지합니다.
- 선택 과제로 GitHub Codespaces 전용 계정을 만들어 **향후 실습 환경**을 표준화합니다.

## Prerequisites

- Edge InPrivate 또는 Chrome 시크릿 창처럼 **프라이빗 브라우저 세션**을 사용할 수 있어야 합니다.
- 본인 명의의 **휴대폰 번호와 결제 수단**이 준비되어 있어야 본인 확인 단계를 통과할 수 있습니다.
- LinkedIn 프로필에 사용할 **기본 인적 정보(이름, 거주 국가, 직무)**를 미리 정리합니다.
- 실습 안내 이미지(스크린샷)를 참고할 수 있도록 **해당 가이드 문서**를 별도 창에서 띄워둡니다.

## Architecture overview

온보딩 플로우는 다음과 같이 서로 연결됩니다.

1. **Outlook 계정**: 모든 후속 서비스의 루트 ID가 되며, 보안·결제 알림이 이 메일로 발송됩니다.
2. **LinkedIn 계정**: Founders Hub 심사에서 필수 항목이며, 회사/아이디어 정보를 보완합니다.
3. **Founders Hub**: Azure 크레딧, 개발 도구, 멘토링 등 스타트업 지원 혜택을 제공하는 포털입니다.
4. **Azure 구독**: `$1,000 크레딧`이 실제로 적용되는 실행 환경으로, Cost Management에서 잔액을 추적합니다.

각 단계는 독립적으로 보이지만 동일한 전용 계정으로 로그인해야 크레딧이 정상적으로 연결되므로,
브라우저 세션과 인증 수단을 일관되게 유지하는 것이 중요합니다.
