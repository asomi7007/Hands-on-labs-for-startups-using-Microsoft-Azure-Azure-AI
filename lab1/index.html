    Lab 1 · Microsoft for Startups Founders Hub & Azure 크레딧 받기¶

    Microsoft for Startups Founders Hub의 Get started quickly 온보딩은 스타트업이 Azure 리소스를 실험해 볼 수 있도록 즉시 사용할 수 있는 $1,000 크레딧을 제공합니다. 이 문서는 신규 계정 생성부터 크레딧 활성화, 확인까지의 모든 단계를 정리했습니다. 브라우저 캡처에 맞춰 단계별로 따라 하면 약 30분 내에 온보딩을 마칠 수 있습니다.

    문서 구조 안내¶

    랩 1 콘텐츠는 한 개의 페이지에 모든 정보를 몰아넣기보다, 실습 흐름에 맞춰 여러 문서로 나뉘어 있습니다. 필요한 주제를 빠르게 찾아볼 수 있도록 아래 페이지를 참고하세요.

    • Introduction: 목표, 선행 준비물, 온보딩 흐름의 큰 그림을 소개합니다.
    • Development: 계정 생성과 크레딧 활성화까지의 실습 절차를 세부 단계로 안내합니다.
    • Deployment: Azure 구독과 크레딧이 제대로 연결되었는지 검증하는 체크리스트를 제공합니다.
    • Security: MFA, 결제 정보 보호, 계정 관리 팁 등 안전하게 크레딧을 운영하는 방법을 정리했습니다.
    • Resources: 공식 문서, 설치 가이드, 환경 변수 등 실습에 도움이 되는 자료 모음입니다.
    • Conversation: 참가자 문의 대응을 위한 대화 흐름과 지원 채널 설계를 다룹니다.
    • Summary: 완료 후 점검해야 할 항목과 다음 단계 추천 작업을 제공합니다.

    Microsoft for Startups Founders Hub란?¶

    Founders Hub는 단순한 크레딧 증정 이벤트가 아닌, 아이디어 구상 → 제품 개발 → 시장 진출(GTM) 전 과정을 지원하는 Microsoft의 스타트업 전용 플랫폼입니다. 계정당 단계별 혜택이 제공되며, Azure 크레딧과 더불어 GitHub Enterprise, Microsoft 365, 멘토링, 기술 지원 등을 순차적으로 받을 수 있습니다.

    무엇이 달라지나요?

    • 클라우드 리소스: 즉시 사용할 수 있는 $1,000 Azure 크레딧으로 MVP를 빠르게 검증할 수 있습니다.
    • 전문가 지원: AI, GTM, 투자 등 다양한 분야의 Microsoft 전문가 멘토링을 예약할 수 있습니다.
    • 도구 생태계: GitHub Copilot, Visual Studio, Azure AI 등 스타트업 운영에 필요한 툴을 할인 또는 무료로 제공합니다.

    사전 점검¶

    기존 개인/업무용 Microsoft 계정에 영향을 주지 않도록, 온보딩 전용 계정과 브라우저 세션을 반드시 준비합니다.

    • 프라이빗 브라우저 세션 사용: Edge InPrivate 또는 Chrome 시크릿 창에서 모든 절차를 진행합니다. 기존 쿠키나 로그인 정보 충돌을 피할 수 있습니다.
    • 전용 Microsoft 계정 생성: 새로운 Outlook 메일 주소를 만들어 진행합니다. 기존 계정의 구독/결제 정보와 섞이지 않습니다.

    단계별 가이드¶

    1. Microsoft 계정 만들기¶

    1. 프라이빗 창에서 Outlook 가입 페이지를 열고 무료 계정 만들기를 선택합니다.
    2. 사용할 이메일 주소(@outlook.kr 등)와 비밀번호를 입력합니다.
    3. 개인정보 수집·이용 안내를 확인한 후 동의하고, 이름/생년월일/국가 등 기본 정보를 입력해 계정을 생성합니다.
    4. 가입이 완료되면 자동 로그인됩니다. "로그인 상태 유지"를 선택해 두면 이후 단계가 수월합니다.

    2. LinkedIn 계정 만들기¶

    Founders Hub 심사에서 LinkedIn 프로필이 필수로 요구됩니다.

    1. LinkedIn 가입 페이지를 열고 앞 단계에서 만든 Outlook 이메일을 입력합니다.
    2. 이름, 거주 국가, 직무 정보 등을 입력합니다.
    3. Outlook 사서함으로 전송된 인증 코드를 확인해 LinkedIn 계정을 활성화합니다.

    3. Founders Hub Get started quickly 신청¶

    1. Founders Hub 가입 페이지에 접속합니다.
    2. Get started quickly 카드에서 Sign up with Azure를 클릭하고, 새로 만든 Microsoft 계정으로 로그인합니다. 로그인 방식은 반드시 비밀번호를 선택합니다.
    3. 2단계 인증을 위해 신뢰할 수 있는 이메일 주소를 등록하고 보안 코드를 입력합니다.
    4. Windows Hello PIN 설정 화면이 나오면 사용 안 함을 선택해 건너뜁니다.
    5. "로그인 상태 유지"를 허용하고 Founders Hub 대시보드로 이동합니다.

    4. Azure 크레딧 활성화¶

    1. Founders Hub Benefits 화면에서 $1,000 Azure 크레딧 카드의 지금 사용 버튼을 클릭합니다.
    2. 사용자 프로필 정보를 입력합니다. 이름, 이메일, 전화번호 등 필수 항목을 정확히 입력하세요.
    3. 결제 수단을 등록합니다. 본인 확인용으로 소액(약 1,000원)이 일시 승인되지만 즉시 취소되며 실제 과금은 발생하지 않습니다.
    4. 약관에 동의한 뒤, 계정 보호를 위한 다단계 인증(MFA)을 설정합니다.
    5. 설정이 완료되면 Azure 포털 환영 화면과 함께 우측 상단에 $1,000 Azure 크레딧 배너가 표시됩니다.

    5. 크레딧 적용 확인¶

    크레딧 배너가 보이더라도 구독 화면에 반영되기까지 최대 1시간 정도 소요될 수 있습니다.

    1. Azure 포털에 로그인해 우측 상단 크레딧 정보를 확인합니다. 만료일과 남은 금액이 함께 표시됩니다.
    2. 좌측 메뉴에서 비용 관리 + 청구 → 결제 방법을 열어 기본 결제 수단에 Azure 크레딧이 등록되었는지 확인합니다.
    3. 비용 관리 + 청구 대시보드의 "남은 크레딧" 카드에서 잔액이 $1,000(또는 환율 적용 금액)으로 표시되는지 확인합니다. 반영되지 않았다면 30~60분 뒤 새로고침합니다.

    6. GitHub 전용 계정 준비 (선택)¶

    랩 실습에서는 GitHub Codespaces를 사용하므로 전용 GitHub 계정을 준비하면 편리합니다.

    1. GitHub 가입 페이지에 접속해 Outlook 이메일과 동일한 비밀번호로 계정을 만듭니다.
    2. 이메일 인증과 보안 퍼즐을 완료한 뒤 로그인합니다.
    3. Codespaces 사용 시 자동으로 동일한 개발 환경을 제공하므로 실습 중 환경 차이로 인한 오류를 최소화할 수 있습니다.

    확인 체크리스트¶

    • Founders Hub Benefits에서 $1,000 Azure 크레딧 활성화 완료
    • Azure 포털 우측 상단에 크레딧 배너 노출
    • 비용 관리 + 청구에서 결제 방법이 Azure 크레딧으로 표시
    • 필요한 경우 GitHub Codespaces 전용 계정 준비

    문서 상단의 번호 태그는 원본 가이드의 스크린샷 순서를 의미합니다. 캡처 이미지는 별도 자료로 제공되므로 단계별 확인이 필요할 때 아래 표를 참고하세요.

    구간 스크린샷 ID 주요 내용

    LinkedIn 가입 #010 ~ #015 회원 가입, 기본 정보 입력, 이메일 인증 흐름

    자주 묻는 질문(FAQ)¶

    Azure 무료 평가판 자격이 없다고 표시됩니다.

    Founders Hub 크레딧은 일반 Azure 무료 평가판과 별개의 프로그램입니다. 기존 평가판 자격과 무관하게 $1,000 크레딧을 받을 수 있으며, 구독이 종량제로 표시되어도 문제 없습니다. 실제 과금 전에 크레딧이 먼저 소진됩니다.

    크레딧이 활성화됐는지 확실하지 않습니다.

    Azure 포털 우측 상단 배너와 비용 관리 + 청구 → 결제 방법 화면을 확인하세요. 크레딧 표시까지 최대 1시간 정도 지연될 수 있으니 충분히 기다린 뒤 새로고침합니다.

    이전에 사용한 전화번호나 카드로 다시 신청해도 되나요?

    가능합니다. 동일한 전화번호, 카드로도 신규 계정에서 크레딧 발급이 확인되었습니다. 다만 브라우저 프라이빗 모드로 새 Outlook 계정(@outlook.kr)으로 로그인했는지 다시 확인하세요.

    크레딧이 전혀 보이지 않을 때는 어떻게 하나요?

    1) 로그인 계정이 새 Outlook 계정인지 확인 → 2) 로그아웃 후 프라이빗 창에서 재로그인 → 3) Founders Hub Sign up with Azure 절차를 다시 진행합니다. 그래도 안 되면 계정 정보, 오류 메시지, 포털 화면 캡처를 첨부해 admin@eldensolution.kr로 문의합니다.

    크레딧 활용 팁¶

    • 크레딧 사용량을 주기적으로 확인하고 불필요한 리소스는 즉시 정리합니다.
    • Azure Cost Management 알림을 활용해 예산 초과를 방지합니다.
    • 실습 및 테스트 리소스는 사용 후 정지 또는 삭제하여 크레딧을 효율적으로 사용합니다.

    이제 $1,000 Azure 크레딧을 활용해 실습 프로젝트를 마음껏 진행해 보세요!
